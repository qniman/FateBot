# Модуль «Контракты»

Модуль для сервера Vespucci (GTA5RP.COM): заявки на роль «Контракты» через модальное окно в Discord, подсчёт суммы навыков, тикеты на одобрение и выдача роли после ознакомления с правилами.

## Настройка

1. Скопируйте `config.example.json` в `config.json` (или отредактируйте существующий `config.json`).
2. Заполните ID каналов и ролей в `config.json`:
   - **channelIds.rolesContract** — канал заявок; бот при старте отправит туда сообщение с кнопкой «Подать заявку». После отправки заявки здесь создаётся сообщение и ветка с суммой навыков.
   - **channelIds.rulesContracts** — канал с правилами; бот при старте отправит туда сообщение с кнопкой «Я ознакомился с правилами».
   - **channelIds.adminReview** — канал для ручной проверки, если не удалось посчитать навыки.
   - **channelIds.approvalTickets** — канал тикетов с кнопками «Выдать доступ к правилам» и «Отказать».
   - **channelIds.logs** — канал логов: кто ознакомился с правилами и получил роль.
   - **roleIds.contractsPending** — роль доступа к каналу с правилами.
   - **roleIds.contracts** — роль «Контракты» (после ознакомления с правилами).

3. Опционально:
   - **maxSkillLevel** (по умолчанию 5) — значения 6–10 считаются как 5; допустимый ввод только 0–10.
   - **minSumForAutoApprove** (по умолчанию 30) — при сумме ≥ 30 бот сам выдаёт доступ к правилам (роль **contractsPending**).

## Как подать заявку

1. В канале **roles-contract** нажать кнопку **«Подать заявку»**.
2. Открывается **6 шагов модальных окон** (по 5 полей в каждом, кроме последнего — одно поле). В каждом поле — отдельный навык или номер паспорта (шаг 1): номер паспорта, Сила, Стрельба, Кулинария, Рыболовство → далее Охота, Поиск сокровищ, … → в конце «Наличие клуба Epsilon» (Да/Нет). Значения навыков: число от 0 до 10.
3. После последнего шага в канале создаётся сообщение «Заявка от @участник. Номер паспорта: …» и **ветка** к нему с суммой навыков и списком уровней. Дальше — автоодобрение или тикет для администратора.

## Какие поля считаются

В поле «Навыки» бот ищет строки вида **«Название: число»**. Список полей: Сила, Стрельба, Кулинария, Рыболовство, Охота, Поиск сокровищ, Фермерство, Строитель, Шахтер, Грузчик, Таксист, Дайвер, Инкассатор, Водитель автобуса, Механик, Пожарный, Дальнобойщик, Курьер, Почтальон, Подрядчик, Мотоклуб, Rednecks, Car Meet, Marrywether, **Наличие клуба Epsilon** («Да» = 1).

**Правила значений:** только числа 0–10; >10 или текст в значении = 0; 6–10 считаются как 5.

## Поведение

- **Сумма ≥ 30:** бот автоматически выдаёт роль **contractsPending**, на сообщение заявки ставится ✅, в канал тикетов уходит «Заявка автоодобрена» (без кнопок).
- **Сумма < 30:** на сообщение ставится ⏳, в канал тикетов создаётся тикет с кнопками «Выдать доступ к правилам» и «Отказать».
- Если навыки не распознаны — тикет на ручную проверку в **adminReview**.
- Администратор нажимает «Выдать доступ» / «Отказать» — на сообщение заявки ставится ✅ или ❌, при выдаче доступа пользователю выдаётся **contractsPending**.
- В канале **rules-contracts** пользователь нажимает «Я ознакомился с правилами» — снимается **contractsPending**, выдаётся роль **Контракты**, запись в **logs**.

## Интенты бота

В ядре бота должны быть включены: `GuildMessages`, `MessageContent`, `GuildMembers`.
