# Модуль «Инактив»

Учитывает активность участников сервера (сообщения, реакции, заходы в голосовой) и по интервалу отправляет отчёт: топ-3 самых активных и топ-3 самых неактивных.

## Настройка

1. Скопируйте `config.example.json` в `config.json`.
2. Заполните:
   - **reportChannelId** — ID канала, куда бот будет отправлять отчёты.
   - **reportIntervalHours** — интервал в часах (например, 24 = раз в сутки).
   - **guildId** — (опционально) ID гильдии для учёта. Если не указан, используется GUILD_ID из конфига бота.

## Учитываемые действия

- Написание сообщения в текстовом канале
- Постановка реакции на сообщение
- Заход в голосовой канал

Действия ботов не учитываются. Данные хранятся в SQLite (файл `inactive.db` в папке модуля).

## Команда

- **/inactive-report** — вывести отчёт по активности за текущий период в ответ (ephemeral).

## Зависимости

В проекте должны быть включены интенты: `GuildMessageReactions`, `GuildVoiceStates` (они добавлены в ядро при наличии модуля).
